사용법: /create-monthly {월} — 월별 YAML 파일 생성/업데이트

- `{월}` (필수): 생성할 월. YYYY-MM 형식 (예: 2026-02)

예시:
- `/create-monthly 2026-02` → 2026년 2월 YAML 파일 생성
- `/create-monthly 2026-03` → 2026년 3월 YAML 파일 생성

인자: $ARGUMENTS

## 동작 방식

1. 사용자에게 다음 중 하나를 선택하도록 질문:
   - **이전 달 복사**: 이전 달(예: 2026-01.yaml) 파일을 복사해서 수정
   - **새로 작성**: 빈 템플릿부터 시작
   - **마크다운 테이블에서 변환**: 사용자가 제공한 테이블을 YAML로 변환

2. 선택에 따라:

### 옵션 1: 이전 달 복사
- 이전 달 YAML 파일(예: `monthly/2026-01.yaml`)을 읽음
- 새 달 파일명(예: `monthly/2026-02.yaml`)으로 복사
- 사용자에게 수정할 항목을 대화형으로 질문:
  - 계좌별, 종목별 금액 변경사항
  - 새로운 종목 추가/삭제
  - 매수기준일 변경
- 수정사항 반영 후 파일 생성

### 옵션 2: 새로 작성
- 기본 템플릿 구조 제시:
```yaml
purchase_day: 26

accounts:
  - name: "계좌명"
    type: "계좌타입"  # ISA, IRP, 연금저축, 일반
    broker: "증권사"
    fee: 0.0015
    holdings:
      - name: "종목명"
        ticker_mapping: "티커"
        amount: 금액
        asset_type: "STOCK"  # STOCK, BOND, CASH
```
- 사용자에게 계좌 및 종목 정보를 순차적으로 입력받음
- 입력 완료 후 파일 생성

### 옵션 3: 마크다운 테이블에서 변환
- 사용자가 제공한 마크다운 테이블을 파싱:
  - 계좌명, 종목명, 금액, 비중을 추출
  - 티커 매핑 자동 추론 (SPY, QQQ, EWY, IEF, GLD 등)
  - asset_type 자동 분류 (국채/SGOV → BOND, 적금/청약 → CASH, 나머지 → STOCK)
- YAML 구조로 변환 후 파일 생성

## 파일 생성 후

다음 안내 메시지 출력:
```
✅ monthly/{월}.yaml 파일이 생성되었습니다!

다음 단계:
1. 파일 내용 확인: monthly/{월}.yaml
2. 분석 실행: /analyze {월}
3. 전체 파이프라인: /run-monthly {월}
```

## 주의사항

- 파일이 이미 존재하면 덮어쓰기 전 확인 요청
- 티커 매핑이 불확실한 경우 사용자에게 확인
- 총 금액 합계를 계산해서 사용자에게 표시
- target_ratio는 자동 계산되므로 YAML에 포함하지 않음
